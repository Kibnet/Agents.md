# Onboarding Template: AGENTS Override

## Когда применять

- Нужно добавить локальные правила репозитория-потребителя поверх центрального каталога.

## Когда не применять

- Если локальная специфика отсутствует.
- Если правило ослабляет MUST из центрального каталога.

## MUST

- Локальные правила могут только ужесточать требования.
- Запрещено отменять или смягчать MUST из центрального каталога.
- Правила должны быть короткими и привязанными к особенностям конкретного репозитория.

## SHOULD

- В начале файла перечислить, какой профиль центрального каталога выбран по умолчанию.
- Для каждой локальной нормы указать причину (инфраструктура, стек, CI, безопасность).

## MAY

- Добавлять project-specific команды сборки и тестирования.

## Команды

```markdown
# AGENTS Override (local)

Базовые правила: `<AGENTS_ROOT>\AGENTS.md`

Дополнительные локальные MUST:
- MUST: использовать только `pnpm` вместо `npm`.
- MUST: запускать `pnpm lint && pnpm test` перед завершением задачи.

Дополнительные локальные SHOULD:
- SHOULD: запускать e2e только при изменениях UI.
```
