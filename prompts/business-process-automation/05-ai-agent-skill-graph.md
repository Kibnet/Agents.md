# Prompt Template: AI Agent Skill Graph

## Purpose

Построить Mermaid `graph TD` граф навыков ИИ-агента по целевому `TO-BE` процессу.

## Required Input

- Mermaid `TO-BE` диаграмма процесса или эквивалентное структурированное описание целевого процесса

## Output Contract

- Верни только Mermaid-код.
- Используй только формат `graph TD`.
- Не добавляй пояснительный текст до или после кода.
- Узлы должны описывать именно способности агента, а не роли людей, статусы или административные шаги.
- Для каждого узла укажи:
  - название навыка;
  - что делает;
  - вход;
  - выход;
  - тип: `Automation Skill` или `Cognitive Skill`.
- Финальный вывод допускается только после успешного прогона через Mermaid lint/validator.
- Если Mermaid lint/validator нашел ошибку, автоматически исправь диаграмму и повторяй проверку, пока диаграмма не станет корректной.
- Если Mermaid lint/validator недоступен, не считай шаг завершенным и не выдавай непроверенный Mermaid-код как готовый артефакт.

## Prompt

Ты - архитектор ИИ-систем и эксперт по функциональному моделированию. По описанию `TO-BE` процесса построй skill graph ИИ-агента в формате Mermaid `graph TD`.

Что должен отражать граф:
- какие функции агент должен уметь выполнять;
- как они декомпозируются;
- где есть ветвления, параллельные ветки и циклы повторной обработки;
- где нужна консолидация результатов.

Общие правила:
- не выноси цель процесса в отдельный узел;
- не используй роли людей как узлы;
- не копируй sequence-логику механически: обобщай ее в способности агента;
- если элемент процесса описывает статус или передачу ответственности, преобразуй его в функциональный навык агента.

Иерархия навыков:
- отрази верхнеуровневые функциональные области;
- разложи их на подфункции;
- опускайся до атомарных задач, если дальнейшая декомпозиция теряет практический смысл.

Формат узла:

```text
"<b>Название навыка</b><br/>Что делает: ...<br/>Вход: ...<br/>Выход: ...<br/>Тип: Automation Skill|Cognitive Skill"
```

Правила связей:
- связи должны отражать только декомпозицию, функциональную последовательность, ветвление, fan-out/fan-in и циклы;
- не используй связи как передачу документов между ролями.

Правила для `subgraph`:
- используй `subgraph` только для верхнеуровневых функциональных областей;
- для `subgraph` применяй технические ID с префиксом `SG`;
- для узлов используй отдельные технические ID с префиксами вида `A1`, `B2`, `C3`;
- не допускай совпадения ID между `subgraph` и node;
- не создавай повторяющиеся ID и самоссылки.

Правила проверки Mermaid:
- обязательно используй Mermaid lint/validator перед финальным выводом;
- предпочитай официальный syntax validation через `mermaid.parse(...)`;
- если он недоступен, используй доступный render-check через `@mermaid-js/mermaid-cli` (`mmdc`) или эквивалентный Mermaid validator;
- если validator вернул ошибку, исправь диаграмму по тексту ошибки и повторяй цикл `lint -> fix -> relint` до успешного результата;
- не выводи Mermaid-код пользователю, пока проверка не завершилась успешно.

Перед выводом проверь:
- код начинается с `graph TD`;
- граф остается именно skill graph, а не process flow;
- отражены ветвления, параллельность и циклы, если они есть в `TO-BE`;
- все типы навыков указаны как `Automation Skill` или `Cognitive Skill`;
- Mermaid lint/validator завершился успешно.
