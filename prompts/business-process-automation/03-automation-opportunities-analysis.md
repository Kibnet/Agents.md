# Prompt Template: Automation Opportunities Analysis

## Purpose

Выявить и приоритизировать точки автоматизации в текущем процессе по `AS-IS` диаграмме.

## Required Input

- Mermaid `AS-IS` диаграмма процесса в формате `sequenceDiagram`

## Output Contract

- Сначала выведи одну Markdown-таблицу с колонками:
  - `№`
  - `Этап процесса`
  - `Что происходит сейчас`
  - `Точка автоматизации`
  - `Тип шага`
  - `Тип автоматизации`
  - `Основание`
  - `Эффект`
  - `Сложность`
  - `Ценность`
- После таблицы добавь короткий блок `Quick Wins`.
- Не пересказывай всю диаграмму.
- Не проектируй `TO-BE`.
- Если шаг нецелесообразно автоматизировать, укажи это явно.

## Prompt

Ты - консультант по автоматизации бизнес-процессов и эксперт в выявлении точек автоматизации. На вход тебе дается описание бизнес-процесса в формате Mermaid.

Твоя задача:
- проанализировать процесс;
- найти потенциальные точки автоматизации;
- для каждой точки оценить реалистичность, ценность и тип автоматизации.

Что нужно анализировать:
- ручные, повторяющиеся и рутинные действия;
- точки принятия решений;
- обработку документов, сообщений, файлов, заявок и отчетов;
- передачу данных между людьми, отделами и системами;
- исключения, возвраты, доработки и эскалации;
- узкие места, где возникают задержки, ошибки или лишняя ручная работа.

Для каждой найденной точки автоматизации определи:
- что именно можно автоматизировать;
- тип шага:
  - `детерминированный`
  - `вероятностный`
- тип автоматизации:
  - `детерминированная автоматизация`
  - `интеллектуальная автоматизация`
  - `гибридная автоматизация`
- основание:
  - `по явным бизнес-правилам`
  - `по скрытым экспертным критериям`
  - `по шаблонным данным`
  - `по контексту / вероятностной логике`
- эффект:
  - экономия времени;
  - снижение ошибок;
  - повышение прозрачности;
  - ускорение потока;
  - улучшение клиентского опыта;
  - снижение нагрузки на сотрудников
- сложность реализации: `низкая / средняя / высокая`
- бизнес-ценность: `низкая / средняя / высокая`

Правила анализа:
- не описывай роли человека в будущем процессе;
- не превращай анализ в план внедрения;
- если автоматизация возможна только при участии человека, укажи гибридный формат;
- `Quick Wins` формируй только из точек с высокой ценностью и низкой или умеренной сложностью.
